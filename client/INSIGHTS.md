## Codebase Patterns
- [2026-06-19] Run-cost UI shares `src/lib/format.ts` (`formatCost` → "—" for unknown cost, never "$0.00") and the `RunCostBadge` component (`compact` | `withTokens`) — reuse them, don't re-format cost inline; `client/src/components/run-cost-badge`.
- Live run logs use a native `EventSource` per active run (`useRunEvents`); on reload the
  log is rebuilt from the persisted trace (`GET /runs/:id/trace`), with a 4s polling fallback.
- Global errors (network/5xx) toast; 4xx is silent → handled inline as empty states.
- Theme is driven by CSS variables + `data-theme`/`data-density`, not a Tailwind theme config.
- [2026-06-20] The PR-list table card (`pulls/styles.ts` `tableCard`) has `overflow:hidden`, so a row-anchored popover positioned `absolute` gets clipped. Render it `position:fixed` from the trigger's `getBoundingClientRect()` and `createPortal` to `document.body`; also `e.stopPropagation()` on the cell + popover so clicks don't trigger the row's navigate-to-PR `onClick`.
- [2026-06-20] One findings popover serves BOTH the PR-list FINDINGS cell and the PR-detail timeline run rows. Because two features consume it, the shared UI was promoted [2026-06-22] out of the PR-list route into the neutral `src/components/findings/` module: `FindingsFilterPopover` (severity chips + list), `FindingPreviewList`, `SeverityCountBadges`, shared `styles.ts`, plus the pure helpers `countBySeverity`/`visibleFindings` (`helpers.ts`) and `SEVERITY_ORDER`/`LOW_CONFIDENCE_THRESHOLD` (`constants.ts`). Import via `@/components/findings/*`. (`KEY_TO_ACTION` stayed in `FindingsPanel/constants.ts` — it's FindingsPanel-only.) Callers resolve `counts` + `findings` and pass an optional `onPick` (list → navigate to PR; timeline → scroll to the run's accordion). The list cell lazy-loads via `usePrReviews` while open; the timeline already has findings in memory (the PR's reviews, keyed by `run_id`).
- [2026-06-20] Severity tally + filter is a reusable primitive `SeverityFilter` (toggle `Chip`s over `SEV` tokens) shared by the popover and each run's `FindingsPanel`; the filter helper is `visibleFindings(findings, hideLow, activeSeverities?)` (now in `@/components/findings/helpers`). A popover resets its filter to "all on" simply by mounting fresh on open (no key hack), since it only renders while open.
- [2026-06-22] Adding an i18n namespace needs NO wiring: `src/i18n/request.ts` (`loadMessages`) `readdirSync`s `messages/<locale>/` and merges every `*.json` as `{ [filename]: {...} }`. Drop `messages/en/<ns>.json` and use `useTranslations("<ns>")` — done. (Test files import a specific `messages/*.json` by relative path; those stay relative since `messages/` lives outside `src/`, so the `@/` alias can't reach them.)
- [2026-06-22] Modal & Drawer (`vendor/ui/kit`) share one a11y hook `useDialogA11y(onClose)` (`kit/useDialogA11y.ts`): Escape-to-close + Tab focus-trap + focus-restore on unmount. Attach the returned ref to the `role="dialog"` node and give it `tabIndex={-1}` + `outline:none`. `onClose` is read through a ref so focus setup runs once on mount (a changing inline-arrow `onClose` won't steal focus mid-interaction). Popovers like `FindingsFilterPopover` keep their own Escape/outside-click — they are not dialogs.

## Tooling
- [2026-06-22] Client ESLint (`eslint.config.mjs`) bridges `eslint-config-next@15` via `FlatCompat` (`@eslint/eslintrc`) because v15 is still eslintrc-style — the native flat export only lands in eslint-config-next@16. Pin `eslint@^9` (config-next@15 isn't built for eslint 10). The TS import resolver pulls native `unrs-resolver`, whose build must be allowed: `pnpm approve-builds` (added to `pnpm-workspace.yaml` `allowBuilds`) — otherwise `pnpm lint` dies in the deps-status pre-check with `ERR_PNPM_IGNORED_BUILDS`. Import boundaries (feature privacy + no deep-relative / `@/app/**`) are enforced by `import/no-restricted-paths` + `no-restricted-imports`; `pnpm lint` runs `--max-warnings 0`.
