## Codebase Patterns
- [2026-06-22] `useRunEvents` (`lib/hooks/reviews.ts`) intentionally depends on `const key = runIds.join(",")`, NOT the `runIds` array ‚Äî its `eslint-disable react-hooks/exhaustive-deps` is deliberate. This makes the SSE subscription content-addressed: callers may pass a fresh array each render (same ids ‚áí no re-subscribe), so do NOT "fix" the disable or memoize `runIds` at call sites.
- [2026-06-19] Run-cost UI shares `src/lib/format.ts` (`formatCost` ‚Üí "‚Äî" for unknown cost, never "$0.00") and the `RunCostBadge` component (`compact` | `withTokens`) ‚Äî reuse them, don't re-format cost inline; `client/src/components/run-cost-badge`.
- Live run logs use a native `EventSource` per active run (`useRunEvents`); on reload the
  log is rebuilt from the persisted trace (`GET /runs/:id/trace`), with a 4s polling fallback.
- Global errors (network/5xx) toast; 4xx is silent ‚Üí handled inline as empty states.
- Theme is driven by CSS variables + `data-theme`/`data-density`, not a Tailwind theme config.
- [2026-06-20] The PR-list table card (`pulls/styles.ts` `tableCard`) has `overflow:hidden`, so a row-anchored popover positioned `absolute` gets clipped. Render it `position:fixed` from the trigger's `getBoundingClientRect()` and `createPortal` to `document.body`; also `e.stopPropagation()` on the cell + popover so clicks don't trigger the row's navigate-to-PR `onClick`.
- [2026-06-20] One findings popover serves BOTH the PR-list FINDINGS cell and the PR-detail timeline run rows. Because two features consume it, the shared UI was promoted [2026-06-22] out of the PR-list route into the neutral `src/components/findings/` module: `FindingsFilterPopover` (severity chips + list), `FindingPreviewList`, `SeverityCountBadges`, shared `styles.ts`, plus the pure helpers `countBySeverity`/`visibleFindings` (`helpers.ts`) and `SEVERITY_ORDER`/`LOW_CONFIDENCE_THRESHOLD` (`constants.ts`). Import via `@/components/findings/*`. (`KEY_TO_ACTION` stayed in `FindingsPanel/constants.ts` ‚Äî it's FindingsPanel-only.) Callers resolve `counts` + `findings` and pass an optional `onPick` (list ‚Üí navigate to PR; timeline ‚Üí scroll to the run's accordion). The list cell lazy-loads via `usePrReviews` while open; the timeline already has findings in memory (the PR's reviews, keyed by `run_id`).
- [2026-06-20] Severity tally + filter is a reusable primitive `SeverityFilter` (toggle `Chip`s over `SEV` tokens) shared by the popover and each run's `FindingsPanel`; the filter helper is `visibleFindings(findings, hideLow, activeSeverities?)` (now in `@/components/findings/helpers`). A popover resets its filter to "all on" simply by mounting fresh on open (no key hack), since it only renders while open.
- [2026-06-22] Adding an i18n namespace needs NO wiring: `src/i18n/request.ts` (`loadMessages`) `readdirSync`s `messages/<locale>/` and merges every `*.json` as `{ [filename]: {...} }`. Drop `messages/en/<ns>.json` and use `useTranslations("<ns>")` ‚Äî done. (Test files import a specific `messages/*.json` by relative path; those stay relative since `messages/` lives outside `src/`, so the `@/` alias can't reach them.)
- [2026-06-22] Modal & Drawer (`vendor/ui/kit`) share one a11y hook `useDialogA11y(onClose)` (`kit/useDialogA11y.ts`): Escape-to-close + Tab focus-trap + focus-restore on unmount. Attach the returned ref to the `role="dialog"` node and give it `tabIndex={-1}` + `outline:none`. `onClose` is read through a ref so focus setup runs once on mount (a changing inline-arrow `onClose` won't steal focus mid-interaction). Popovers like `FindingsFilterPopover` keep their own Escape/outside-click ‚Äî they are not dialogs.

- [2026-06-24] When pushing a conditionally-rendered `Button` to the far end of a flex toolbar, use an unconditional `<div style={{flex:1}} aria-hidden />` spacer rather than `marginLeft:auto` on the button ‚Äî vendored `Button` doesn't guarantee `style` forwarding (see kit prop-forwarding note below), and the spacer stays correct even when the button is absent; `ConventionsListView/styles.ts` (grow).
- [2026-06-22] kit inputs forward arbitrary DOM props INCONSISTENTLY: `TextInput`/`Textarea` spread `...rest` onto the real `<input>`/`<textarea>` (so `aria-*`, `id`, `data-*` work), but `SelectInput`/`SearchableSelect` do NOT (internal element, no passthrough). `FormField` wires `aria-describedby` to its hint via `cloneElement` on the child ‚Äî it applies on TextInput/Textarea and is a harmless no-op on the selects. To attach DOM/a11y attrs to a select, extend that component first.
- [2026-06-23] The shared `Markdown` (`vendor/ui/primitives/Markdown.tsx`) styles ONLY inline elements unless you add block renderers: the design reset zeroes h1‚Äìh4/p margins and Tailwind preflight flattens heading font-size/weight, so markdown headings/lists render as plain body text until you supply explicit `h1/h2/h3` + `ul/ol/li` component overrides.
- [2026-06-23] To persist a UI reorder/edit across unmount, write the mutation's authoritative response into the cache via `qc.setQueryData(key, data)` ‚Äî `invalidateQueries` alone loses it, because leaving the page cancels the background refetch and the remounted component re-seeds from STALE cache (even though the DB write succeeded); `lib/hooks/skills.ts` (useSetAgentSkills).
- [2026-06-24] `Modal` (`vendor/ui/kit/Modal.tsx`) body pads via the `bodyPad` prop (default `"20px 24px"`); it previously had ZERO body padding ‚Äî unlike sibling `Drawer`, whose body is `padding:24` ‚Äî so naive modals sat flush to the rounded edge while header/footer were inset 24. Modals that manage their own body (already self-padded, or full-bleed like a search bar + scrollable `pre`) MUST pass `bodyPad={0}` to avoid double-padding (`PromptBlock` full prompt view, `CreateAgentModal` whose `styles.ts` `s.body` already pads).
- [2026-06-24] Per-category skill generation (`CreateSkillFromConventionsModal` per-category mode) creates a skill per category by looping `POST /skills` client-side ‚Äî and `skills.name` has NO unique constraint server-side (`server/src/db/schema/knowledge.ts` skills table), so re-running on the same repo silently creates DUPLICATE skills. This is accepted/out-of-scope by the multi-skill plan (server atomicity + 409 uniqueness deferred), not a bug ‚Äî don't "fix" it client-side; de-dup belongs in a future server bulk route.
- [2026-06-24] `prevRunningRef` + `useEffect([running])` (single dep) is the correct pattern for detecting SSE stream close (`running: true‚Üífalse`) without a race condition; using `[running, jobId]` as deps fires the effect prematurely on the render where `jobId` is first set before `useSseEvents` can call `setRunning(true)`; `client/src/lib/hooks/sse.ts` (useSseEvents), `client/src/app/conventions/_components/ConventionsListView/ConventionsListView.tsx`.
- [2026-06-26] The merged Intent block (intent + RISK AREAS in ONE card) has ONE Recompute that fires the two mutations SEQUENTIALLY ‚Äî `await recomputeIntent.mutateAsync()` THEN `recomputeRisks.mutateAsync()`, NOT `Promise.all`. The server `analyzeRisks` reads the PERSISTED intent (`repo.getIntent`) to anchor risk scope, so parallelizing would recompute risks against the STALE intent. Do not "optimize" to parallel. Combined aria-live derives from both mutations (computing if either pending; `recomputeFailed` if either errors; `briefUpdated` when both succeed); `IntentCard.tsx` (handleRecompute).
- [2026-06-26] `SmartDiffViewer` JOINS THREE sources by file `path`: `usePrSmartDiff` (role grouping + `finding_lines` + `split_suggestion`), `usePullDetail().files` (the `patch` text ‚Äî the `SmartDiff` contract carries NONE), and `usePrReviews` (per-line SEVERITY ‚Äî `finding_lines` are bare line numbers with no severity). Inline severity tints AND the "N findings" badge breakdown come ONLY from `usePrReviews`; keep the join pure + memoized in `helpers.ts`; a file present in the groups but missing a patch (binary) still renders header+badge with an empty body (`parsePatch(null) === []`); `client/src/app/repos/[repoId]/pulls/[number]/_components/SmartDiffViewer/helpers.ts` (joinSmartDiff).
- [2026-06-26] `SectionLabel` (`vendor/ui/primitives/SectionLabel.tsx`) lays out `icon + children + right` in ONE `alignItems:center` flex row ‚Äî so asymmetric padding on the children (e.g. a header `<span>` with `padding:"12px 16px 0"`) pushes the text down while the icon stays put, leaving the icon floating ABOVE the text. Inset such a header by padding a WRAPPER `<div>` around `<SectionLabel>`, never the children span; `SmartDiffViewer.tsx` (GroupCard).
- [2026-06-26] `FindingsFilterPopover` (`components/findings/`) has a card-mode: pass `renderContent?: ReactNode` and it drops the `SeverityFilter` chips + preview list (KEEPS the title+close header) and renders the injected node in a scrollable `s.cardBody` (width 480 vs the 400 list default). This lets the SmartDiff inline-tag path show full **route-private** `FindingCard`s (with `useFindingAction` Accept/Dismiss) WITHOUT the shared popover importing route code ‚Äî the route composes, the shared module stays a dumb `ReactNode` slot (keeps the `shared ‚Üí app` dependency direction legal). Do NOT promote `FindingCard` into `components/` (it's a `prReview`-domain feature component). `SmartDiffViewer.tsx` (FindingCardStack), `FindingsFilterPopover.tsx`.
- [2026-06-26] The findings popover captures its `findings` in state at OPEN time (`{anchor, findings, key}`). For the inline-tag card stack to reflect Accept/Dismiss IN PLACE + auto-close when empty, render the stack from the LIVE `findingsAtLine.get(lineNo)` (recomputed each render from `file.findings`), NOT the captured snapshot ‚Äî otherwise the `useFindingAction` mutation invalidates `["reviews", prId]`, `joinSmartDiff` re-filters dismissed findings, but the open stack (bound to the stale snapshot) won't shrink/update. Auto-close via a `useEffect` when the live line list hits 0 (guarded to the card path so the badge path never auto-closes). `SmartDiffViewer.tsx` (FileRow).
- [2026-06-27] The global `@media (prefers-reduced-motion: reduce)` rule zeroes `animation-duration` on `*` (`vendor/ui/styles.css`), which FREEZES functional spinners too (Button-loading `RefreshCw`, `Loader2`, skeleton shimmer, live `ddpulse` dots) when the OS has "reduce motion" on ‚Äî not just decorative motion. CSS can't select by keyframe name, so re-enable essential indicators via a marker class (e.g. `.dd-spin`/`.dd-pulse`) exempted with `!important` inside the media query; `client/src/vendor/ui/styles.css`, `primitives/Button.tsx`.
- [2026-06-27] `FindingsFilterPopover` clamps only the HORIZONTAL position (`left`), not vertical: `top = anchor.bottom + 6` with no flip/clamp, so an anchor near the viewport bottom pushes the `position:fixed` panel off-screen and the page can't scroll it (internal `cardBody` scroll doesn't help ‚Äî the panel's top is already too low). Clamp/flip vertically (measure height via `useLayoutEffect`) like a real popover; `client/src/components/findings/FindingsFilterPopover.tsx`.
- [2026-06-28] A de-emphasis theme token can't double as badge TEXT: the `orphaned` anchor badge was `color:var(--stale)` on `bg:var(--text-muted)` (gray-on-gray, ~0 contrast). Fix needs a tint token `--stale-bg` AND a separate readable `--stale-text` (‚â•4.5:1) ‚Äî `--stale` itself is too pale for text; `content_changed` reuses the readable `--warn`/`--warn-bg` pair with a DISTINCT LABEL (WCAG: distinguish states by text, not color alone); `client/src/app/repos/[repoId]/pulls/[number]/_components/FindingCard/FindingCard.tsx` (ANCHOR_BADGE), `client/src/vendor/ui/styles.css`.
- [2026-06-28] The client imports `@devdigest/shared` contracts TYPE-ONLY (zero value-imports), so the contracts' `zod` schemas are tree-shaken OUT of the client runtime bundle even though `zod` is a client dep. DON'T value-import a contract schema to derive a constant (e.g. `SkillType.options`) ‚Äî it would be the first runtime contract import and pull zod into the client bundle; instead guard an enum-shaped constant with a TYPE-level check (`{‚Ä¶} satisfies Record<SkillType, true>` + `Object.keys`); `client/src/lib/skills.ts` (SKILL_TYPE_VALUES).
- [2026-06-30] For a file reference that may point OUTSIDE the PR diff (e.g. blast-radius CALLERS ‚Äî repo files with no diff row, so in-app `?tab=diff` scroll silently no-ops), deep-link to github.com via `githubBlobUrl` (repoFullName from `useActiveRepo`, head_sha from `usePullDetail` ‚Äî NOT the response contract) rendered through `MonoLink` (new-tab `<a>`, degrades to plain text when repo/sha unknown). Changed-symbol files link file-level (no line in the blast contract); callers link `#L{line}`; `BlastCard.tsx`, `lib/github-urls.ts` (githubBlobUrl).
- [2026-06-30] Styling Mermaid nodes under `securityLevel:"strict"`: CSS custom properties (`var(--‚Ä¶)`) are UNRELIABLE inside the sanitized SVG, so `classDef` must use LITERAL HEX ‚Äî and since the graph is hardcoded `theme:"dark"`, the dark-token hex values are the correct, theme-stable choice. A React-rendered legend that lives OUTSIDE the SVG can still use CSS vars; keep the two palettes in sync by hand; `BlastCard.tsx` (buildMermaid, LegendDot).

## Tooling
- [2026-06-24] Client component tests use `fireEvent` from `@testing-library/react` ‚Äî `@testing-library/user-event` is NOT a dependency, so `import userEvent` fails typecheck. Pattern: `vi.mock` the data hooks + `next/navigation`, render under `NextIntlClientProvider` (messages imported by relative path, `@/` can't reach `messages/`) + `ToastProvider`; assert toasts via their rendered text. See `AgentEditor.test.tsx`, `FindingCard.test.tsx`, `CreateSkillFromConventionsModal.test.tsx`.
- [2026-06-22] Client ESLint (`eslint.config.mjs`) bridges `eslint-config-next@15` via `FlatCompat` (`@eslint/eslintrc`) because v15 is still eslintrc-style ‚Äî the native flat export only lands in eslint-config-next@16. Pin `eslint@^9` (config-next@15 isn't built for eslint 10). The TS import resolver pulls native `unrs-resolver`, whose build must be allowed: `pnpm approve-builds` (added to `pnpm-workspace.yaml` `allowBuilds`) ‚Äî otherwise `pnpm lint` dies in the deps-status pre-check with `ERR_PNPM_IGNORED_BUILDS`. Import boundaries (feature privacy + no deep-relative / `@/app/**`) are enforced by `import/no-restricted-paths` + `no-restricted-imports`; `pnpm lint` runs `--max-warnings 0`.
- [2026-06-26] jsdom has no `Element.prototype.scrollIntoView` ‚Äî testing a click-to-jump / scroll-to-line affordance THROWS unless you stub it (`beforeEach: el.scrollIntoView = vi.fn()` or on the prototype); assert the spy was called rather than real scrolling; `SmartDiffViewer.test.tsx`.
- [2026-06-28] jsdom 25 ships NO `window.PointerEvent`, so RTL `fireEvent.pointerDown/Move/Up` dispatches a bare `Event` with no `clientX/clientY` (and React's `pointerdown` listener may not fire) ‚Äî drag/pointer tests silently no-op. Polyfill it in `beforeEach` as `class extends MouseEvent { pointerId }` and assign to `window`+`globalThis` `PointerEvent`; also `getBoundingClientRect().height`/`offsetHeight` are 0, so stub the panel height + `window.inner*` to exercise viewport-clamp math; `FindingsFilterPopover.test.tsx`.
- [2026-06-28] `eslint.config.mjs` derives `ROUTE_FEATURES` (the `import/no-restricted-paths` zone list) DYNAMICALLY via `readdirSync(<config-dir>/src/app)` (directories only, minus `[`/`_`/`.`), NOT a hand-maintained array ‚Äî a manual list had silently dropped `skills`+`conventions`, leaving them unguarded as cross-feature importers. Don't "simplify" it back to a static list; new top-level features get guarded automatically; `client/eslint.config.mjs`.
- [2026-06-30] mermaid@11 reads graph font size ONLY from `themeVariables.fontSize` (a CSS string, e.g. `"18px"`), NEVER the top-level numeric `MermaidConfig.fontSize` (it's in the schema but the render pipeline never reads it). To make a large graph scrollable instead of shrink-to-illegible, also set `flowchart.useMaxWidth:false` (keeps the SVG's intrinsic `width`/`height`, drops `max-width`) and wrap it in a fixed `maxHeight` + `overflow:auto` viewport; `client/src/components/mermaid-diagram/MermaidDiagram.tsx` (MermaidDiagram).
